# Public VN Structure Reference

This public file keeps reusable structural lessons without naming or quoting private reference works.

## Opening Patterns

- Problem-first school-life opening: start from a small rule failure, object, errand, or relationship interruption before formal identity exposition.
- Abnormal-event opening: let a strange device, message, sound, or contradiction create a question before the premise explains itself.
- Ritual/conflict opening: make identity visible through obligation, routine, danger, or a task that cannot be refused.
- Place-with-guide opening: use a room, threshold, guide figure, or missing memory to create immediate orientation pressure.
- Slice-of-life undercut opening: establish ordinary group texture, then let one detail behave wrong.

## Route / Chapter Lessons

- Common route should seed unresolved tensions instead of only introducing cast.
- Character routes should reinterpret common-route details, not merely add private scenes.
- True route or final route should require understanding built by previous routes, not only flag collection.
- Branches may reconverge, but they must leave visible state: object, line variant, relation debt, knowledge, or route lock.

## Dialogue Lessons

- Dialogue can run in blocks; avoid mechanical A/B/A/B when one speaker owns pressure.
- Same-speaker runs work when each click adds a new tactic: dodge, tease, corner, confess sideways, repair, or retreat.
- Comedy needs pressure: a nickname, unpaid debt, awkward object, social rule, or deadline.
- Mature emotional scenes often speak around the taboo word through errands, rituals, work tasks, and object handling.

## Interiority Lessons

- First-person thought should be a private inference or choice cost, not author commentary.
- Put thought after pressure accumulates, then return to visible action before it becomes an essay.
- Relaxed thought may use fillers, half jokes, self-corrections, and retreating particles if they belong to the character.

## Production Lessons

- Every click should alter screen state, information, relationship pressure, choice pressure, or branch memory.
- Every choice needs immediate feedback and delayed callback.
- Exports are build artifacts; CSV remains source of truth.
