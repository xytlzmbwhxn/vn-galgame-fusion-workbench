# Complete Work Method

## Diagnosis

The previous workbench could produce a valid VN script table, but a valid table is not yet a complete work. A complete work needs pressure that survives across scenes:

- a protagonist who wants the wrong thing for understandable reasons
- a professional ritual that forces choices into concrete actions
- an opening question and a costly answer for every scene
- a moral pressure that changes what the player thinks the system is listening for
- side characters who bring the world in, not only support the mystery

## Sources Turned Into Method

- Scene structure: use a scene-level question and answer, plus goal, conflict, outcome.
- Subtext: characters talk about safe objects while the unsafe want moves underneath.
- Game choice: the player must be able to plan and later see what the game remembered.
- Branch-and-bottleneck: branches return to the main spine with altered state, not erased state.
- Four-layer narrative: gameplay action, character action, plot event, and theme pressure must touch.
- VN pacing: character bonding should run at the character's dialogue pace, not at plot-summary pace.

## Rain Gate Method

Every key scene in `rain_gate_demo` must pass this contract:

| Layer | Requirement |
| --- | --- |
| Scene Question | A concrete question raised by the opening screen. |
| Professional Ritual | A station-work action: registration, lost property, door lock, camera check, handover, incident report. |
| Character Lie | What the POV character believes that makes the correct action hard. |
| Opposition | A person, object, timer, or rule that refuses the easy version. |
| Dramatic Turn | New information that makes the original plan unsafe. |
| Sequel Decision | A reactive choice after the turn, not a neutral menu. |
| State Memory | Relationship, evidence, promise, object, or deadline changes. |
| Theme Pressure | The scene tests whether records protect people or expose them. |

## Writing Procedure

1. Write the scene question in one line.
2. Pick the station ritual that forces people to touch objects.
3. Define what each person says they want and what they are using that want to hide.
4. Draft the professional action first, then let dialogue attach to the action.
5. Add one interruption that changes the next five lines.
6. Place the choice after the interruption.
7. Write branch consequences as changed objects, not explanations.
8. End on an object or missing record that will matter later.

## Prose Standard

The desired prose is not decorative. It should feel lived-in:

- names of cheap objects
- work fatigue
- small social debts
- wrong words said to avoid dangerous ones
- local human mess inside formal procedure
- humor that appears because someone is tired, not because the script wants a joke

Cut any sentence that explains what the scene has already made visible.
