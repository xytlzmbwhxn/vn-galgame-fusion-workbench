# Branch And Bottleneck With State

## Call This When

The project needs meaningful choice without exploding into full separate routes.

## Procedure

1. Let branches differ in action, information, relationship cost, or object ownership.
2. Reconverge only after the branch has changed state.
3. Write a bottleneck line or object that reads differently depending on branch state.
4. Record the callback target before drafting the branch.

## Output Fields

- branch-specific gain
- branch-specific wound
- reconvergence beat
- later callback target

## Failure Signs

- branches say different words but leave the same state
- reconvergence erases what happened
- the player cannot tell the choice was remembered
- branches feel like skipped alternate paragraphs
