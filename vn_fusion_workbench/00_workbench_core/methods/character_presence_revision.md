# Character Presence Revision

## Purpose

Revise a VN draft without sanding away character identity. This method belongs to the
workbench and does not require any outside original file at generation time.

## Call This When

- The user says the text is flat, generic, AI-like, too stiff, or not cute/lively enough.
- A speaker loses voice when the name box is hidden.
- The draft sounds polished but has no relation cost.
- A rewrite has changed rows that need final cleanup.

## Three Checks

- Position: speaker role, body state, relation, object, and immediate job are visible.
- Cost: the line risks trust, time, pride, route state, evidence, debt, or safety.
- Handprint: the speaker keeps their rhythm, particles, punctuation, dodge, and quirk.

## Procedure

1. Mark changed row ids.
2. Diagnose one failure per row.
3. Change the smallest possible unit.
4. Prefer action, object, and relation callbacks over abstract explanation.
5. Keep character-owned roughness unless it blocks reading.
6. Run `changed_line_voice_integrity_check` before delivery.

## Output Contract

The revision note must include changed row ids, row-level reason, revised text, and
memory update status.
