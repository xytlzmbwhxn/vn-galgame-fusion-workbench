# Choice Mental Model

## Call This When

Before writing a menu, consequence, route flag, or bad end.

## Core Use

The player must learn what the game is listening for. Hidden consequences are allowed; random-feeling consequences are not.

## Procedure

1. Name the axis the choice tests: proof, trust, time, debt, safety, suspicion, route.
2. Make both options gain and lose something.
3. Show immediate feedback in a line, object, expression, or available option.
4. Store delayed feedback in `effects` and state delta.
5. Pay it off later through changed text, missing option, altered route, or bad end.

## Output Fields

- choice axis
- immediate feedback
- delayed callback
- variable or flag

## Failure Signs

- one option is obviously correct
- the choice label describes morality instead of action
- no one reacts differently
- the variable never changes future prose
