# Literary Depth And Liveliness Control

## Call This When

Load before every serious Chinese Galgame / AVG draft or rewrite when the user says:

- characters are not vivid, cute, or human enough
- psychology stops too early or feels mechanical
- prose has low literary quality
- dialogue sounds like a lesson, a report, or an AI summary
- the scene has rules but lacks spirit, pressure, or emotional aftertaste

## Source Calibration

This card uses public, reusable theory only as craft calibration:

- Aristotle, Poetics: reversal and recognition are converted into playable scene turns.
- Stanford Encyclopedia of Philosophy, Personal Identity: identity pressure is converted into choices that threaten a character's self-story.
- Dialogic voice / heteroglossia theory: multiple social languages are converted into character-specific speech engines.

Do not quote theory in drafts. Convert it into object handling, dialogue pressure,
thought pockets, choices, and branch debt.

## Core Diagnosis

A weak VN line usually fails before style. It fails because the line has no lived position. The speaker is not holding a job, a lie, a debt, a body state, an object, and a risk at the same time.

Before adding prettier words, restore the pressure stack:

role in scene -> thing in hand -> social mask -> private want -> unsafe cost -> next click effect

If any layer is missing, the line becomes flat even when the sentence is smooth.

## Philosophy Layer For Actual Drafting

These are writing operations, not essay topics.

### 1. Tragedy / Comedy Turn

Use reversal and recognition as scene engines:

- reversal: a practical action changes meaning after one detail appears
- recognition: a character understands a relation, debt, or self-deception through an object or reply

For VN writing, let the player click through: ordinary task -> small mistake -> joking cover -> private cost -> choice -> changed object.

### 2. Dialogic Voice

Each character carries a social language. A clerk, a repeat student, a manager, a classmate, and a parent should have different ways of hiding embarrassment, asking for help, refusing debt, and ending a sentence.

Voice is not vocabulary alone. It includes who names objects, who uses procedure words, who jokes before a request, who answers the previous line, who changes address terms, and who can leave a sentence open.

### 3. Narrative Identity

A character is the story they keep telling to survive the day. A good choice pressures that story.

Ask before writing a line:

- What version of themselves are they protecting?
- What would this click make harder to keep pretending?
- What small action proves the self-story has cracked?

### 4. Ethics As Play

The theme must become repeated player behavior. If the theme is care versus procedure, the player should repeatedly handle receipts, names, deposits, umbrellas, records, callbacks, and visible debt. No line may deliver the theme as a lecture when a menu, object, or state flag can make the player perform it.

## Liveliness + Half Step

The project preference is fixed: every character should be slightly more actable, playful, and memorable than the plain archetype.

This does not make every character noisy. Each character receives one extra live hook:

- an object nickname system
- a private scoring habit
- an oddly specific complaint
- a tiny vanity
- a wrong but charming theory
- a procedural tic that becomes tenderness
- a repeated address term that changes meaning

The live hook must create pressure, not decoration.

## Psychological Depth To The End

Thought rows should continue until they create a next action. Stopping at a labeled feeling or a clean understanding is too shallow.

Use this chain:

trigger -> first wrong guess -> private cost -> avoided name -> practical worry -> reply/action

Thought must stay inside the protagonist''s pressure. It may be small, crooked, and unfinished. It should not sound like a critic labeling a scene.

## Cute / Casual Speech Control

Casual charm comes from timing and ownership, not random particles.

Use playful language when the speaker is covering shame, buying time, asking for help without lowering pride too much, testing whether the other person will play along, or making an object carry the embarrassing topic.

Do not make all speakers equally cute. Assign separate engines:

- object-naming character: nicknames, mock ceremony, soft complaints, fast self-correction
- procedure character: forms, receipts, columns, clipped replies, dry understatements
- authority character: short warnings, accounting words, practical threats

Cute must not become self-display. If a joke, nickname, or odd comparison only
shows that the writer can be cute, cut it or push it into a cost:

- it must hide shame, ask for help, test trust, dodge thanks, or change a relation
- it must invite the other character to act, refuse, correct, or remember
- it should become less shiny when the character is tired, scared, or exposed
- it should not stack three similar object jokes in a row unless the excess itself is the pressure

When a character is lively, keep one half-step of awkwardness. A line that is too
neat reads like performance; a line with a small overreach, retreat, or practical
consequence reads like a person.

## Sentence And Click Density

A click can be short only when it changes pressure. A click can be long only when the player needs to stay trapped in that breath.

Before delivery, mark important rows with one click function: reply pressure, new information, character leakage, private inference, relation cost, object state change, choice pressure, or breath control.

Low-information fragments must merge. Overpacked lyric rows must split at the exact point where inference or pressure changes.

## Branch And Memory Depth

Visible choices are only surface. A branch is real when it leaves persistent debt: who owes whom, who keeps proof, which address term becomes available, what object is stored or missing, which later line changes, or which memory becomes easier or harder to trigger.

Every accepted branch must update runtime state only after the draft is accepted.

## Pre-Draft Checklist

- Literary pressure
- Scene reversal
- Recognition point
- Character self-story under threat
- Live hook + half step
- Dialogue ownership
- Thought-pocket chain
- Cute/casual timing
- Branch debt
- Forbidden AI habits to scan

## Blocking QA

Reject or repair the draft when:

- a character can be recognized only by name box
- the live hook appears once and stops affecting pressure
- thought labels emotion without creating a next action
- the scene''s idea is stated more clearly than it is played
- playful punctuation is evenly sprinkled across speakers
- a branch reconverges without later text, object, or relationship debt
- the prose uses balanced contrast or summary logic to sound clever
- the scene could lose all choices and still read the same
