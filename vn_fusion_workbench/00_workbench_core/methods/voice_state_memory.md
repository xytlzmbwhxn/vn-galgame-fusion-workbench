# Voice State Memory

## Call This When

Before drafting dialogue and after completing a scene.

## Inputs

- character card
- runtime character state
- promises and debts
- scene summary
- recent state delta

## Procedure

1. Read the character card for baseline voice.
2. Read runtime state for current axes and active fears.
3. Choose the highest axis and make it alter line length, object handling, or address.
4. Use at least one promise, debt, or old object in the scene.
5. After the scene, update runtime state and memory entries.

## Output Fields

- current axis driver
- allowed sentence shapes
- forbidden drift
- promise or debt used
- memory update target

## Failure Signs

- the character speaks the same in every scene
- backstory appears only as explanation
- a promise is made but never tracked
- runtime state exists but the dialogue ignores it
