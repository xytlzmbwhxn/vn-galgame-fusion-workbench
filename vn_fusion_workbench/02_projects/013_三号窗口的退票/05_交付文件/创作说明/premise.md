# 《三号窗口的退票》项目前提

Genre / Tone: 现实微悬疑，夜班服务窗口，轻口吻下压着旧案。

Player Role: 林桥，城际车站三号退改签窗口夜班 clerk。

Core Promise: 玩家通过核票、查班次、盖章、留票角等窗口操作，决定一名陌生女孩能拿走哪一种可被承认的痕迹。

Central Pressure: 23:58 关窗前，孟雨拿来一张三个月前的车票要求退票。票已过期，班次停运，车站系统显示无效；票角却压着今天的红章。

Primary Mechanic: 退票流程微交互。核票号、查章、比对班次、盖拒退章、登记异常。

Secondary Mechanic: 证据留存与信任值。玩家每次偏离流程都会让 `station_risk` 上升，也让孟雨交出更多隐情。

Reading Mode: ADV，限制视角阅读，一格文本框推进一个可演出的动作或一次话语攻防。

Target Length: 首部小短篇，单场景约 80 行，可导出 CSV、可读稿、Excel 与多引擎草稿。

Route Structure: 单场景分支后回收，保留状态差异。选择让玩家体验“按章”与“偏心”都要付代价。

Replay Value: 两个选择分支影响最终备注、票角去向、孟雨对林桥的称呼，以及后续路线变量。

Visual Motifs: 雨水糊掉的票面、窗口玻璃的反光、退票失败章、热敏打印小票、监控盲区的红灯。

Audio Motifs: 雨棚滴水、打印机走纸、广播误报、印章敲桌、末班车怠速。

Ending Philosophy: 手续有时伤人，有时也能把人暂时托住；关键在于写手续的人愿不愿意承担签名。
