# Public Style Handprint

这个文件是公开仓库用的安全风格手印。它不记录具体参考来源，只保留工作台需要执行的写作动作。

## 生成时必须体现

- 开头先给环境、异常物件、行动压力或问题，不用“我叫某某/我是某某”的硬自我介绍开场。
- 第一人称受限视角只写主角能看见、听见、推断和误读的东西。
- 心理行要像角色此刻真的会冒出来的念头，而不是作者解释。允许“嗯……”“好吧”“大概”“先这样”等放松、缓冲和自我修正。
- 对白块不要机械 A/B/A/B。短促顶嘴可以轮流，长段要有同说话人 run、物件打断、误读、称呼变化或心理口袋。
- 每个点击单位必须让玩家多知道一点、多误会一点、多承担一点，或者看到屏幕状态变化。
- 选择必须留下可回调状态：物件位置、关系债、信息差、路线 flag、时间压力或后续台词差异。
- 废萌/日常不是无压力。压力可以很小：一瓶温掉的汽水、一张不能读的纸条、一个昵称、一句没登记的笑。

## 避免

- 用主题总结替代行动。
- 把心理写成“他其实是在……”这类全知解释。
- 用固定对称句式解释“不是 X，而是 Y”。
- 为了文学感堆光、雨、命运、沉默等通用意象。
- 把外部参考原句、角色名、专有场景或可识别桥段带进新稿。

## 可用句法动作

- 先写屏幕/物件，再让主角用一句松一点的心理反应。
- 让角色把害羞藏进工作流程、售后、登记、实验、赔偿、复核等小制度。
- 让一句玩笑兼具关系试探和下一步任务。
- 在选择前插一小段私心，但私心必须挂在物件、动作或风险上。
