# 项目结构总览

`VN_Workbench_Project` 是建议上传 GitHub 的仓库根目录。

## 顶层

```text
VN_Workbench_Project/
├─ README.md                      GitHub 首页说明
├─ START_HERE.md                  人和 AI 的启动入口
├─ GITHUB_UPLOAD_GUIDE.md          GitHub 页面填写和 push 步骤
├─ 项目结构总览.md                 这份结构说明
├─ .gitignore                     依赖、runtime、中间产物忽略规则
├─ skills/                        顶层 portable skills，方便直接看到
├─ AI_HANDOFF_PACKAGE/             AI 接手包、状态和镜像
├─ external_refs/                  可公开的外部参考说明
└─ vn_fusion_workbench/             正式工作台
```

## skills

```text
skills/
├─ portable_skill_manifest.json
├─ galgame-workbench-loader/
├─ vn-fusion-writer/
├─ vn-scene-drafting/
├─ vn-presence-revision/
└─ vn-character-memory/
```

这是给人和 AI 最容易发现的技能入口。工作台内部还保留同一批 skills：

```text
vn_fusion_workbench/00_workbench_core/portable_skills/
AI_HANDOFF_PACKAGE/portable_skills/
```

三处都要能通过 `vn_skill_backup_check.py` 校验。

## AI_HANDOFF_PACKAGE

```text
AI_HANDOFF_PACKAGE/
├─ handoff_notes/                  当前状态、操作规则、失败预防
├─ manifests/                      portable manifest
├─ portable_skills/                skills 镜像
├─ prompts/                        起草提示模板
├─ references/                     可公开安全参考层
├─ skills_snapshot/                快照说明
└─ source_ledgers/                 来源账本说明
```

它的目标是：另一个 AI 不看聊天记录，只看文件，也能知道现在该怎么启动、该读哪些方法、该交付哪些内容。

## vn_fusion_workbench

```text
vn_fusion_workbench/
├─ 00_workbench_core/
├─ 01_reference_log/
├─ 02_projects/
├─ 06_学习输入/
├─ README.md
└─ .gitignore
```

### 00_workbench_core

```text
00_workbench_core/
├─ docs/              工作流、路径协议、公开生成流程
├─ methods/           可复用写作方法卡
├─ portable_skills/   工作台内部 portable skills
├─ references/        公开安全参考层
├─ schemas/           JSON / CSV 结构约束
├─ templates/         角色卡、场景卡、剧本表模板
├─ tools/             工作台命令行工具
└─ runtime/           只提交 README 和 path_aliases.json
```

可复用规则必须进入 `methods/`、`docs/`、`tools/` 或 `portable_skills/`，不能只留在聊天记录里。

### 02_projects

每篇作品都是独立项目。标准结构：

```text
02_projects/<project_id>/
├─ 00_project_memory/
│  ├─ bible/
│  ├─ cards/characters/
│  ├─ runtime_state/characters/
│  └─ memory_logs/character_memory/
├─ 01_narrative_design/
│  ├─ routes/
│  ├─ scenes/
│  └─ novel_architecture/
├─ 02_generated_content/
│  ├─ scripts/csv/
│  ├─ scripts/excel/
│  └─ drafts/readable/
├─ 05_交付文件/
│  ├─ 可读稿/
│  ├─ 剧本表_CSV/
│  ├─ 剧本表_Excel/
│  ├─ 引擎导出/
│  └─ 角色设定/
└─ 99_内部工作/
```

目录边界：

- `00_project_memory/`：角色卡、运行状态、关系记忆、故事专属设定。
- `01_narrative_design/`：scene card、route map、互动设计、小说工程层。
- `02_generated_content/`：生成稿、CSV 源剧本、可读预览。
- `05_交付文件/`：最终给人看的可读稿、CSV、Excel、引擎导出、角色设定。
- `99_内部工作/`：QA、deep audit、context trace、style contract、临时分析。

### 06_学习输入

```text
06_学习输入/
├─ 教程文本/
├─ 视觉小说剧本文本/
├─ _风格画像/
├─ _处理日志/
└─ _外部项目评估/
```

公开仓库保留学习流程、模板和可安全分享的抽象风格资产。真正用于生成的公开风格资产放在 `_风格画像/`，例如：

```text
06_学习输入/_风格画像/VN文风手印融合资产_20260623.md
06_学习输入/_风格画像/style_profile_cn_gal_tutorial_lessons_20260623.md
```

## 完整流程

1. 读 `START_HERE.md`。
2. 读顶层 `skills/`。
3. 运行 `vn_context_bootstrap.py`。
4. 读取项目记忆、角色卡、runtime state、memory log、scene card、route map。
5. 生成或重写 CSV。
6. 渲染可读稿。
7. 导出 WebGAL / Ren'Py / Ink / Yarn / Godot Dialogue。
8. 导出角色设定。
9. 跑 validate / deep-audit / quality-debt / encoding / delivery guard / handoff guard。
10. 跑 workspace cleanup。
11. 再提交 Git。
